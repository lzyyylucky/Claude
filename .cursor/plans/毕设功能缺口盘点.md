# cct-cn 系统功能缺口与待完善项（面向「代码生成系统」产品）

说明：此前版本偏重论文交付物表述；此版改为**纯产品与功能视角**：在当前已实现能力之外，系统仍可补强哪些功能。

---

## 已有主线（简要）

| 模块 | 能力 |
|------|------|
| CLI | `init`、`template-list`、中文 `generate`，Markdown 模板驱动写文件（[`src/main.cpp`](src/main.cpp)，[`templates/zh`](templates/zh)） |

以下均在上述之外的 **缺口或可改进点**。

---

## 一、产品线闭环缺口（优先级一般偏高）

### 1. CLI 与 Web 未在同一界面闭环

- **现状**：命令行 `generate` 与浏览器对话两套入口；站内不负责枚举 CLI [`templates/zh`](templates/zh)，也未提供「按同名模板一键 CLI 生成」。
- **可完善**：Web 展示可读模板清单并列调用后端封装的一次生成接口（需在服务端封装 [`generator`](src/generator) / [`run_generate`](src/main.cpp) 逻辑）；或至少文档化「两条路径何时用哪一个」。

### 2. 两套「模板源」未互通

- **现状**：CLI 用 `templates/zh`；站内 Agent/Skill 用 [`component-templates`](component-templates) + 用户目录；无导入/导出、无仓库级同步。
- **可完善**：从「官方 component-templates」批量导入为用户组件；或「导出为 CLI 可用 `.md` 文件」。

### 3. 嵌入式 Web IDE 与聊天工作区耦合度一般

- **现状**：[`ide-app`](ide-app) 独立 Node 构建，产物 `ui/ide/ide.js`，需手工 build + `ui-sync`。
- **可完善**：发布流程脚本一键化；或在「设置」里提示 IDE bundle 是否过期。

---

## 二、会话与工作区功能缺口

### 4. 会话生命周期不完整

- **现状**：有清空当前线程消息 [`POST /api/chat/clear`](src/web/server.cpp)；线程列表侧不一定提供「删除整条会话记录」「重命名标题」「搜索历史」等。
- **可完善**：按线程删除（删除 threads 元数据 + 历史）、改标题、列表检索。

### 5. 工作区能力边界

- **现状**：文件 API；会话锚点；diff 预览写盘。
- **可选**：Git 状态展示 / 一键提交；大目录列表分页；拖拽上传文件夹（若浏览器与安全允许）。

### 6. 聊天附件类型单一

- **现状**：文本为主；工作区通过快照文件带入上下文。
- **可选**：图片等多模态输入（依赖模型能力与上传存储）。

---

## 三、运营与安全类缺口（商用必补，毕设可选）

### 7. 模型调用治理缺失

- **现状**：配置里有 API Key、`max_tokens` 等；无全局速率限制、无按用户配额、无调用次数/费用统计面板。
- **可完善**：简单计数器 + 限额中间件；失败重试策略配置。

### 8. 传输与安全加固未系统化

- **现状**：本地 HTTP 演示常见场景。
- **可完善**：HTTPS、反向代理示例、`Secure` Cookie、敏感配置不写前端。

### 9. 自动化测试与 CI

- **现状**：README 未描述单元/集成测试。
- **可完善**：对路径校验、JSON 提取、关键 API 状态码做最小 smoke 测试。

---

## 四、跨平台与部署缺口

### 10. HTTP 服务仅限 Windows 构建路径

- **现状**：[`serve`](src/main.cpp) 在 `#ifdef _WIN32` 内；非 Windows `run_http_server` 为空实现。
- **可完善**：若题目要求「可部署服务器」，需移植 [`server.cpp`](src/web/server.cpp) 所用 WinSock 等到跨平台栈或拆一层抽象。

---

## 建议实施优先级（仅功能）

| 优先级 | 方向 |
|--------|------|
| P1 | 会话：删除/重命名/搜索线程（若产品上要像一个完整 IDE Copilot） |
| P1 | CLI ↔ Web 模板关系的要么打通要么文档边界清晰 |
| P2 | 用量与限额（防 Key 滥用） |
| P2 | Git 或导出补丁（代码生成系统常见诉求） |
| P3 | 非 Windows 服务端 |
| P3 | 多模态附件 |

---

若你希望下一步把其中某几条落成「具体接口 + UI 改动清单」，可指定优先级区间（例如只做 P1）。
