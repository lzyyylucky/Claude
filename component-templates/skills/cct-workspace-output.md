# 技能：CCT_WORKSPACE 写回约定

当用户消息中包含工作区多文件快照，且你需要修改或新建文件时：

## 必须遵守

1. 在整段回复的**最后**（不要在 Markdown 代码围栏内）追加；若平台区分思考与正文，**至少**要让该块出现在用户最终能看到的**正文**末尾（不要只写在思考里；前端会尽量从思考里兜底解析，但不保证）。

```text
CCT_WORKSPACE:
{"writes":[{"path":"相对工作区根的路径","content":"该文件完整新内容"}]}
```

2. `writes` 为数组：只包含你**新建或修改**的文件；路径使用正斜杠 `/`；`content` 为**完整文件内容**（整文件覆盖）。
3. 不要改动未提及的文件；不要输出二进制或图片的文本伪装。

## 自然语言

回复正文仍用中文向用户说明改了什么、如何验证；`CCT_WORKSPACE` 仅机器写盘用。
