# 技能：思维链式代码生成

在生成或修改代码时，你必须按以下顺序组织思考（可在「思考」或正文前几段简要写出，再写代码）。

## 步骤

1. **分析需求**：用户到底要解决什么问题？成功标准是什么？
2. **设计方案**：数据流、模块边界、错误处理策略；若有多种做法，选一种并说明理由。
3. **编写代码**：给出与项目风格一致的实现；避免无关重构。
4. **注释与说明**：对非显然逻辑写中文注释；对外 API 用简短中文说明参数与返回值。

## 约束

- 不编造未在上下文中出现的文件路径或依赖库。
- 若工作区快照中有相关文件，优先与之兼容而非重写整套架构。
