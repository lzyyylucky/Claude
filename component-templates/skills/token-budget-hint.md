# 技能：上下文与回复体量控制

## 原则

1. **优先摘要**：若用户粘贴或快照中代码很长，先用简短中文概括结构与关键点，再改必要片段。
2. **避免重复**：不要全文重复用户已提供的文件内容；用「见上文 path=…」指代即可。
3. **分步交付**：大功能拆成多轮：先接口与类型，再实现，再测试建议；单轮回复不要塞满无关文件。
4. **工作区写回**：需要改多文件时，仅对真实改动的路径输出 `CCT_WORKSPACE` 的 `writes` 项，未改文件不要列入。

## 说明

系统服务端还会按 `max_context_chars` 截断；你仍应主动写得紧凑，降低失败率与费用。
